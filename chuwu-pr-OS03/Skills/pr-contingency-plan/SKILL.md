# 公关预案

## 定位

预案不是追求零风险。零风险通常不存在。

预案要做的是：在动作上线前，把可能消耗信任、引发误解、被对手放大、被媒体截图、被监管追问的点提前拆开，然后判断哪些要改动作，哪些备口径，哪些只监测。

最终产出三样东西：

- 这次最可能被挑什么。
- 被挑时能公开署名怎么答。
- 上线前哪些地方必须改、必须备、只需看。

## 知识调用

默认按本文件完成预案。遇到高风险动作、责任归因、爆料格式、平台传播、利益相关方复杂或组织取舍问题时，读取 `references/knowledge-map.md`。

知识调用后仍要落到三类处理：必须改动作、必须备口径、只需监测。不要用理论解释替代上线前清单。

## 先定标

不要一上来扫风险。先判断这次动作服务什么：

- 曝光。
- 信任。
- 品类教育。
- 风险修复。
- 行业地位。
- 政策合法性。
- 渠道或用户承接。

不同目标能承受的争议不同。短期声量不能透支长期信任。

## 路由

命中高频场景时，读取对应 reference：

| 场景 | 读取 |
|---|---|
| 品牌出现在综艺、纪录片、直播、联名等内容中 | `references/content-placement.md` |
| 产品上新、升级、换配方、换包装 | `references/product-launch.md` |
| 明星、达人、IP、KOL 合作 | `references/spokesperson-fandom.md` |
| 换厂、换渠道、规则调整、服务政策变化 | `references/change-issue.md` |

如果不属于四类，不要硬套。走通用流程即可。

## 通用流程

```text
定标 → 对象研究 → 历史与同类回查 → 六格风险扫描 → 外部攻击推演 → 口径与处置 → 收口
```

一次动作涉及多类，就叠加相关 reference，但输出仍要收成一份预案。

## 六格风险扫描

每次都从六个角度过一遍，没风险也写「暂无明显风险」，不要漏格：

- **动作本身**：定价、承诺、姿态、价值立场是否招打。
- **产品或内容交付**：产品、服务、内容、活动是否有失败模式。
- **相关方**：用户、员工、合作方、渠道、特定群体是否会觉得被冒犯或受损。
- **规则环境**：法律、监管、平台规则、行业规范有没有红线。
- **历史旧伤**：品牌、行业、合作对象过去的问题会不会被翻出。
- **关联方与对手**：代言人、供应商、平台、竞品、恶意账号是否会把风险传导过来。

## 处理分三类

每个风险最后都要落到三类之一：

- **必须改动作**：事实、产品、承诺、合作对象、利益安排、上线节奏站不住。不能靠话术解决。
- **必须备口径**：动作可以做，但外部会追问，需要可公开、可核验、能一致执行的回答。
- **只需监测**：风险存在但不值得主动解释，设监测点和升级红线即可。

口径标准：

- 能公开署名。
- 对得上事实。
- 不扩大承诺。
- 不把计划写成结果。
- 不用情绪安抚替代事实解释。

## 输出结构

```text
结论
一句话判断这次动作的舆情底色、最大风险和上线建议。

一、动作目标与舆情底色
写这次动作服务什么，外部会按什么逻辑理解它。

二、风险地图
按六格扫描，写具体风险、触发条件、严重程度。

三、外部攻击推演
写媒体、用户、竞品、监管或特定群体最可能怎么质疑。

四、应对口径
一问一答。只写能公开署名、事实兜得住的话。

五、上线前清单
分成必须改动作、必须备口径、只需监测。
```

## 质量红线

- 不用「加强沟通」代替口径。
- 不用口径掩盖事实问题。
- 不把所有风险都写成高风险。
- 不为了全面列一堆无关相关方。
- 不替业务承诺无法确认的结果。
- 不写灰色执行手段。

如果最后需要在上线、延期、换表达、牺牲传播效果、保护信任之间做取舍，转入 `judgment-hub` 拍板。
