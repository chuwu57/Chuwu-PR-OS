# 产品上新 / 升级舆情预案

适用：产品上新、升级、换配方。这类预案独有一个别的活没有的硬动作——合规前置。别的预案守「别人会怎么说你」，产品预案要先守「你自己怎么说自己」：向内那一关没过，向外全是被动。

## 适用分叉
- **全新品类 / 新品**：走完整路径，重在第 3、4 步（品类卖点认知、竞品测绘）。
- **改款 / 升级 / 换配方**：可略过卖点建模，重在第 5、6 步（历史回查、新旧切换）。

## 分步结构

### 1. 产品事实档案
先把产品真相钉死，这是地基。核心卖点、定位、规格与价格、关键成分 / 材料 / 配置、生产或供应方、各类背书（第三方检测、资质、专利、实验数据，及能否在上线前到位）、视觉物料（升级一定做 before/after 对比）、上新节奏与货盘。写法：客观、可核验，每个数字能落回某份凭证。事实兜不住的卖点，回头改产品，别改说法。

### 2. 宣传 claim 合规审查
把品牌想喊的每句卖点过一遍，挑出高危的。四类最该警惕：
- 绝对化用词（「最」「第一」「0」一旦坐实就是把柄）；
- 承诺过满或周期过短（「X 天见效」越短风险越高）；
- 口径直接挂功效 / 结果 / 健康（最招质疑和监管投诉）；
- 无实证的对比（「碾压某爆款」必须有可公示依据）。

写法：用红蓝双色标——红＝带来负面 / 质疑 / 投诉增量，蓝＝利好消费者、稳舆论、减客诉，一眼看清哪句该动。产出是给品牌的**修改建议**（把「0」改成「无明显」、把绝对化收成可证伪的表述），不是维护话术。这一步最该前置，也最对得起诚实——先逼宣传站住，后面才有得守。

### 3. 品类与卖点认知建模（上新重，升级可略）
碰到全新品类 / 新材料 / 新技术，建一份底：原理上为什么好、相对旧方案强在哪、稀缺或壁垒在哪、市面还有谁在做。写法：当内部说明书写，每段落点是「所以差异化 / 风险在哪」。这份底子决定被追问「真有用吗」「凭什么比别人好」时有没有据可依。

### 4. 竞品同品类测绘
选同品类、同价格带、同卖点逻辑的对标，扒五样：声量与负面数据（总量、负面量、负面率、起量节点对应它哪次动作）、卖点与宣传口号、对外页面怎么写、客服怎么答、用户主要骂什么。写法：数据先行，取数逻辑写明（机器判定的负面率标注仅供参考）。对手最常踩的坑，大概率也是你的坑。

### 5. 历史自身回查（升级重，上新可略）
品牌过往几次改款 / 升级 / 换供应方各自引发了哪类舆情，每次落一句「踩在哪个口上」（品牌、配置、价格、渠道、版本不一致）。写法：归类，不堆案例。目的是预判这次大概率复发哪类。

### 6. 风险分维度审查
用固定维度扫这次产品：价格、成分 / 材料 / 配置、品类与品质稳定性、购买体验、使用中的负面反馈、宣传口径、原创 / 抄袭质疑。升级额外挂一组「新旧切换」专项：新旧混发、换货诉求、版本差异、旧款去哪买（评论区最容易被关联旧负面、被顺手推竞品）。写法：每维度一句「判断有无明显风险」加触发点，无风险写「暂无」留痕。

### 7. 异议应答口径准备
把第 6 步每个预判到的质疑，配一条官方站得住的应答口径。是「口径」——客服、官号、详情页三处共用的统一说法，客观、可核验、对得上第 1 步的事实。写法：一问一口径。口径必须能被凭证兜住；兜不住的，回第 1 或第 2 步改产品 / 改宣传，别在这一步用话术圆。

### 8. 落地审核清单
客服话术、产品详情页、上线值班。写法：上线前在三个对外出口（客服界面、详情页、官号）各核一遍——说法是否和事实、和口径一致。

## 重心提醒
产品预案独有的硬动作是第 2 步合规前置。别的预案是「预测风险、备好应对」，产品预案要再往前一步，在准备任何应对之前先把宣传本身改到站得住。风险话术再漂亮，补不了一句立不住的卖点带来的窟窿。一句话收口：别的预案守的是别人会怎么说你，产品预案先守的是你自己怎么说自己。
