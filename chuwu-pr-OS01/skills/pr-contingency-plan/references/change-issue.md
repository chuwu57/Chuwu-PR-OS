# 变更类议题前置预案

适用：业务侧要动一刀——换厂 / 换供应方、换渠道、政策或规则调整。舆情侧提前埋伏。内核是「用知情权换确定性」：把告知做在舆情起来之前，永远比事后澄清便宜。（换配方走产品类 `product-launch.md`。）

## 分步结构

### 1. 事实与触发
讲清到底改了什么、为什么改、哪一点会引爆。写法：客观陈述，先不防御。把「变了什么」和「消费者会怎么感知这个变」分开写。

### 2. 历史同类发酵点归类
品牌或行业里过去同类变更的负面，落在哪几个口上。常见落点：品牌（信任）、配方 / 配置（降级质疑）、价格（性价比）、渠道（版本不一致 / 买不到旧款）。写法：归类，不堆案例，每类一句「为什么会炸」。

### 3. 竞品案例与处置拆解
别家做过同样的事：怎么炸的、又怎么处理的。重点看人家「多走了哪一步」（提前公示？换货政策？转移话题？）。写法：案例 + 它的处置动作 + 这一步是否值得抄。

### 4. 上游风险核查
供应方 / 工厂 / 原料 / 渠道的风险确认——新供应方有无历史负面、社媒上有无既存讨论。写法：这一格决定你敢对外承诺到什么程度，核查不实就别承诺。

### 5. 风险分维度
拿通用风险维度扫一遍，重点压在产品向（配方 / 配置变化）、销售向（价格 / 渠道）、行业向（合规、品类共性质疑）。写法：每类标一句潜在烈度。

### 6. 前置把控手段
核心是把「让消费者有迹可循」做在前面。常用手段：
- **提前公示**——在店铺主页 / 详情页明示变更，保障知情权，让消费者有迹可寻。
- **备好凭证**——新旧版检测报告 / 资质，预估会有人做新旧对比。
- **配料 / 配置稳态**——变更前后关键成分别有大跳变，同步公示来源。
- **客服话术前置**——一定会有人进线问，客服端解决一批，社媒端的负面就少一批。
- **官方科普内容**——上线后铺基础科普，抚平顾虑（公开署名的品牌内容，非伪装 UGC）。

写法：每条手段对应它要灭的那个具体风险，不是泛泛列动作。

### 7. 应对口径
对预判到的每个质疑配官方口径。常见三类角度：转移话题焦点、强调关键指标无变化（凭证背书）、以真实「都用过」视角说明体验稳定。写法：一问一口径，口径必须兜得住凭证。

### 8. 值班与收口
上线节点谁盯、客服 / 详情页 / 官号三处口径是否一致。写法：落成清单。

## 重心提醒
变更类的胜负手在第 6 步——被动应对永远比主动告知贵。能在舆情起来前把公示、凭证、客服话术铺到位，这一仗就赢了大半。把第 3 步「竞品多走了哪一步」当成持续偷师的习惯。
