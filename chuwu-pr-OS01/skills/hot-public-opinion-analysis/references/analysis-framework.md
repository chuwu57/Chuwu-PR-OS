# 舆情分析简明框架

## 1. 是否已经形成舆情

先回答"这件事是否已经从单点信息变成公众态度的集合状态"，再决定是否进入完整研判。

### 舆情的定义

舆情是针对特定话题的群体性意见。若某帖子/报道未形成普遍性话题、未被群体性转发评论，则不构成舆情。

### 判断要素

1. **源头**：最早信息来自哪里——当事人爆料、媒体报道、平台内容、监管公开信息、行业传闻、二手转述。
2. **扩散链路**：是否多平台传播、媒体跟进、KOL二次解读、固定标签或固定话术。
3. **讨论规模**：是否超出小范围圈层，进入行业、社会或政策公共议题。
4. **媒体类型**：主流媒体、行业/财经媒体、地方媒体、自媒体、垂类KOL、社交平台用户、监管/协会/官方账号。
5. **业务相关性**：与本业务、同业、平台、供应链、用户心智、政策环境、商业模式或价值观议题的关联。

### 事实审核（信息搜集后、分析前必须执行）

对搜集到的背景信息逐条核查消息源等级：
- **权威信源**（可直接引用）：企业官方公告/财报、监管公开文件、主流权威媒体署名报道、上市公司公告
- **可信信源**（需注明出处）：知名财经/行业媒体署名报道、行业协会公开数据
- **待核实信源**（须标注"据XX报道，未经官方证实"）：非署名报道、自媒体/博主/KOL发布信息
- **网帖传言**（须标注"网帖传言，真实性待核实"）：脉脉匿名帖、微博普通用户帖、论坛帖、短视频用户内容

规则：不得将网帖传言当既定事实；涉及具体数字须追溯原始出处并标注信源等级；官方口径与网帖有出入时以官方为准。

### 三种结论

- **已形成舆情**：跨平台扩散、媒体/KOL跟进、公共议题化、热榜或权威介入。
- **尚未形成舆情**：单点信息、局部讨论、未经验证传闻。直接给结论，不强行进入完整危机研判。
- **处于舆情前夜**：尚未爆发，但具备被放大、借势、翻旧账或类比本业务的风险。

## 2. 舆情实质分析

按"每一个舆情背后都是利益相关方的博弈"的逻辑，理清争论本质。

### 核心分析逻辑

> **规则：舆情实质分析的重点是回答"这个舆情为什么会出现"，而不是简单罗列各方受损情况。**
>
> 分析路径：先判断"事件在舆论上冒犯了谁"或"谁发起了舆论争议"→再追问背后是哪些社会矛盾、群体对立、价值观冲突引发了集体情绪→最后才落到对各方的影响。
>
> 禁止：把"利益相关方分析"做成简单的受损/受益清单。必须穿透到社会情绪的根源。

### 分析维度

- **事实层**：已知事实、待确认信息、外界推测——分开标注。
- **情绪层**：公众不满/焦虑/同情/愤怒/质疑集中在哪里。
- **舆情成因**（核心）：这个舆情为什么会出现？事件冒犯了谁？谁发起或放大了争议？背后映射的是哪些社会矛盾、群体对立或价值观冲突？为什么此时此刻这件事能引发集体情绪？
- **叙事标签**：是否被提炼为"平台责任""资本逐利""劳动权益""食品安全""算法治理""价格歧视""消费公平""监管缺位"等公共标签。
- **新特点**：与同类事件相比的新证据、新人群、新平台、新标签、新政策背景、新传播路径。
- **长期影响**：对品牌心智、行业认知、政策语境、监管关注、合作伙伴信任或业务决策的影响。

### 深度质疑环节（必须执行）

初步分析完成后，启动"深度质疑"子agent对结论进行挑战：

1. **反驳初步结论**：初步分析是否停留在表面？是否存在更深层的社会结构性矛盾？
2. **理论框架重新解读**：援引1-2个与该舆情最贴合的权威学者理论框架（社会学、传播学、社会心理学、政治经济学等），重新解释"为什么会出现集体情绪"。
3. **反直觉洞察**：提供至少一个穿透表象的深度观点。

对比初步结论与质疑结论后，汇总修订为最终输出。

### 最终输出三点

1. 舆情为什么会出现——穿透表面叙事，说清背后的社会矛盾或群体情绪根源（必须有理论框架支撑）。
2. 本次舆情值得关注的新特点。
3. 对相关方决策、品牌形象或社会情绪的长期影响。

## 3. 舆情走势研判

### 五阶段判断

| 阶段 | 特征 |
|---|---|
| 潜伏期 | 矛盾存在但未被广泛感知；单点案例、零星投诉、小范围讨论 |
| 激化期 | 同类话题规模化出现；多案例串联、KOL/自媒体提炼标签、行业媒体跟进 |
| 爆发期 | 官方主流媒体、监管、重要平台热搜或公共议程介入 |
| 反转期 | 新证据、新时间线、新当事人发声或新叙事使公众判断发生变化 |
| 降温期 | 信息增量减少、热榜退出、媒体不再跟进、评论分散或回应被接受 |

### 趋势判断看什么

- 是否还有关键事实未披露
- 是否存在更多同类案例可被串联
- 是否容易被提炼成简单标签
- 是否贴近当下政策、社会情绪或行业矛盾
- 是否已有监管、主流媒体、权威人士或头部KOL介入
- 当事方是否可能回应、道歉、反击、整改或被约谈
- 是否存在"借势"条件

### 输出原则

不做绝对预测。使用条件化判断："短期看""若出现……则可能升级""若没有新增事实，预计逐步降温"。必要时引用历史类似案例辅助判断。

## 4. 对业务的参考意义

把舆情当成业务演练素材，转化为可执行的自查和沟通建议。

### 重点回答

- **对本业务的影响**：是否会被类比、翻旧账、误伤、追问、截图对比或纳入监管语境。
- **业务风险因子**：安全、价格、履约、算法、劳动者权益、未成年人、老年人、商家权益、数据隐私、内容治理、消费公平、垄断、社会责任。
- **业务自查动作**：查事实、查规则、查历史投诉、查异常数据、查外部话术、查同类案例、查已有预案。
- **待讨论的问题**：是否需要和PRBP、法务、安全、GA等同学讨论；对齐口径、风险阈值、监测范围、回应原则、升级机制或外部沟通边界。

### 底线

不写"加强沟通""优化机制"等空话。每条参考意义指向一个具体业务动作或讨论问题。
