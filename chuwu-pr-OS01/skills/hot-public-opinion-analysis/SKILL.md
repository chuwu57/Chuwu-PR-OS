# 热点舆情分析

## 定位

舆情分析不是热度复述，而是把外部事件转成业务判断。

每次都要回答四个问题：

- 它有没有从单点信息变成群体性意见。
- 它真正冒犯了谁，为什么能被放大。
- 它接下来会升级、降温、反转，还是进入长期议题。
- 它和我们的品牌信任、业务风险、公关资产有什么关系。

默认输出要能直接转发给业务方。不要写学术论文，不堆材料。

## 先做事实核验

先查清楚再判断：

- 源头：当事人、媒体、监管、平台、网帖、截图还是二手转述。
- 扩散：是否跨平台，是否有媒体、KOL、热榜、官方或监管介入。
- 信源等级：官方公告、权威媒体、行业媒体、自媒体、匿名帖、无法溯源截图。
- 关键数字和时间线是否有原始出处。

规则：

- 网帖和自媒体信息只能作为「舆论场内容」，不能直接当事实。
- 当官方口径和网帖冲突，事实以官方为准；舆论反应另写。
- 无法确认就写「待核实」，不要补故事。

## 判断顺序

### 1. 是否已经形成舆情

舆情是针对特定话题的群体性意见，不是单条内容。

分三档：

- **已形成舆情**：跨平台扩散，媒体/KOL 跟进，出现固定标签，进入公共议题或热榜。
- **舆情前夜**：还没爆，但具备被放大、借势、翻旧账、类比本业务的条件。
- **尚未形成舆情**：仍是局部讨论、单点投诉、未经验证传闻。

### 2. 舆情实质

不要只写谁受损、谁受益。先问：

- 这件事冒犯了谁。
- 哪种群体情绪被激活。
- 它被包装成了什么公共标签。
- 为什么是现在发酵。
- 有没有行业旧矛盾、社会情绪或价值观冲突在借题发挥。

输出要穿透表象，但语言要白。可以借理论帮助思考，不要把理论写成正文。

### 3. 走势研判

判断所处阶段：

- 潜伏期：零星信息，矛盾还没被广泛看见。
- 激化期：同类案例串联，标签出现。
- 爆发期：热榜、主流媒体、监管、头部 KOL 介入。
- 反转期：新证据、新主体、新时间线改变判断。
- 降温期：信息增量减少，讨论分散。

看几个升级条件：

- 是否还有关键事实没披露。
- 是否容易被一句话标签化。
- 是否有更多同类案例可串联。
- 是否贴近政策、监管、社会情绪或行业旧账。
- 当事方回应是否可能激化矛盾。

不要做绝对预测。用条件句写清楚：如果出现什么，会怎样；如果没有新增事实，会怎样。

### 4. 对业务的参考意义

最后必须回到业务：

- 我们是否会被类比、误伤、翻旧账或截图对比。
- 是否涉及安全、价格、履约、劳动者、未成年人、老年人、宠物健康、数据隐私、消费公平、社会责任等风险因子。
- 是否影响品牌信任、行业语境、媒体关系、话语体系、内部风险治理。
- 是否需要自查事实、规则、投诉、历史口径、渠道动作或已有预案。

不是所有热点都值得投入资源。没有业务相关性，就明确写「观察即可」。

## 输出结构

```text
一、是否已经形成舆情
给明确分档和一句理由。

二、舆情实质分析
写这件事为什么会被放大，背后的情绪和公共标签是什么。

三、舆情走势研判
写当前阶段、升级条件、降温条件。

四、对业务的参考意义
写和我们有什么关系、需要查什么、是否进入下一步动作。
```

## 质量红线

- 不编造来源、数据、链接、时间线、动机和公众反应。
- 不把热度当影响。
- 不把外部事件硬拉到本业务。
- 不写「加强沟通」「高度关注」这种空话。
- 不替法务、监管、业务直接定责。
- 不把学术概念当输出。
- 不用长表格稀释判断。

如果分析后问题变成「我们要不要回应、自查、做预案、调整动作」，转入 `judgment-hub` 做最终取舍。
