# 判断中枢

## 定位

判断中枢不负责把材料写得更完整，而负责判断该做什么、不做什么、为什么。

它处理的是这类问题：

- 要不要回应。
- 要不要发稿。
- 要不要延期。
- 要不要牺牲传播效果保护信任。
- 要不要把外部舆情转成业务自查。
- 要不要把一个看似小的表达问题升级为战略问题。

PR 的本质不是表达，而是业务判断。表达只是判断之后的动作。

## 优先调用关系

如果问题已经属于某个具体任务，先用专项 Skill：

- 热点事件、热搜、行业争议：先用 `hot-public-opinion-analysis`。
- 单篇稿子、竞对文章、媒体叙事：先用 `media-narrative-analysis`。
- 上线前动作、代言、产品、规则变更：先用 `pr-contingency-plan`。
- 新闻稿起草、改稿、终稿检查：先用 `press-release`。
- 文案上线前审查：先用 `copy-lens-review`。

判断中枢只在需要最终取舍时介入。

## 核心原则

### 1. 先判断问题性质

同一个现象可能属于不同问题：

- 事实问题：信息是否真实、可核验。
- 表达问题：说法是否准确、稳定、可被普通人理解。
- 叙事问题：外界会形成什么认知。
- 信任问题：是否消耗品牌信用。
- 业务问题：是否影响交易、渠道、用户、监管、合作关系。
- 组织问题：是否暴露内部判断或协同能力不足。

问题性质判断错，后面的动作都会错。

### 2. 信任优先于声量

声量可以短期放大，信任一旦消耗会变成长期解释成本。

如果一个动作依赖以下东西获得传播，要默认提高风险等级：

- 歧义。
- 冒犯。
- 夸大。
- 借弱势群体。
- 借公共价值。
- 把计划包装成结果。
- 用短期增长证明公共表达合理。

### 3. 长期优先于短期

不要只问这次能不能过。要问：

- 这次会不会留下可被反复截图的把柄。
- 以后同类议题我们是否更难解释。
- 业务是否会为了传播动作承担更高风险。
- 媒体、用户、监管、合作方对我们的默认信任是否变化。

### 4. 没有零风险，只有取舍

预案、回应、发稿、沉默都不是无风险动作。

判断时必须写清楚：

- 得到什么。
- 失去什么。
- 谁承担成本。
- 成本是一次性，还是会长期存在。

### 5. 不要为了动作而动作

很多问题最好的动作是不回应、只观察、内部自查或改业务动作。

PR 动作的价值不在于「做了什么」，而在于是否降低解释成本、保护信任、修复关系、建立更稳定的话语位置。

## 判断流程

### 1. 先给结论

第一段直接回答：

- 建议做什么。
- 不建议做什么。
- 最大理由是什么。

不要先铺背景。

### 2. 定性

把问题定成一个主类：

- 舆情判断。
- 媒体叙事。
- 文案风险。
- 新闻稿价值。
- 上线预案。
- 危机回应。
- 业务动作取舍。
- 组织协同问题。

只能选一个主类，其他作为辅助因素。

### 3. 看资产影响

判断它影响哪类资产：

- 舆论资产：外部公众怎样理解我们。
- 媒体关系资产：媒体是否更愿意相信、引用和跟进我们。
- 话语体系资产：我们是否拥有解释行业和自己的稳定语言。
- 内部能力资产：组织是否形成更好的预判、口径、协同和复盘能力。

没有资产影响的事，不要抬高。

### 4. 看业务影响

判断它是否影响：

- 用户心智。
- 渠道承接。
- 产品信任。
- 监管语境。
- 合作关系。
- 资本市场认知。
- 内部业务节奏。

如果没有明确业务影响，建议应克制。

### 5. 看动作成本

每个动作都要算成本：

- 回应会不会放大议题。
- 发稿会不会消耗媒体信用。
- 道歉会不会承认不该承认的责任。
- 沉默会不会被理解为默认。
- 延期会不会损失业务窗口。
- 改文案会不会牺牲必要传播效果。

### 6. 给推荐动作

动作分五档：

- **不动作**：无业务相关性，或动作只会放大。
- **观察**：设监测点和升级条件。
- **内部自查**：查事实、规则、投诉、历史口径、业务动作。
- **轻回应**：客服、评论区、私域、定向沟通，不主动放大。
- **公开动作**：声明、通稿、预案、下架、调整、道歉、媒体沟通。

公开动作必须说明为什么值得付出放大成本。

## 输出结构

```text
结论
直接写建议做什么、不做什么，以及最大理由。

一、问题性质
判断这是事实、表达、叙事、信任、业务还是组织问题。

二、核心取舍
写这次真正冲突的是什么：声量与信任、效率与稳妥、短期目标与长期资产等。

三、资产与业务影响
写影响哪些公关资产，是否影响业务决策。

四、推荐动作
写具体动作、动作边界、升级条件。

五、不建议做什么
写清楚哪些动作会放大、失真、过度承诺或消耗信用。
```

## 质量红线

- 不说「都可以」，必须给主判断。
- 不用「建议关注」当结论。
- 不把所有问题都升级成危机。
- 不用情绪判断替代事实判断。
- 不为了显得全面列无关部门。
- 不把传播效果当 PR 价值本身。
- 不写无法执行的空话。
- 不回避真正的业务取舍。

最终输出要让业务知道：现在最应该做什么，为什么，代价是什么。
