# 稿件分析

## 定位

读稿子不是复述它写了什么，而是判断它在替谁建立什么认知。

本 Skill 处理三类问题：

- 这篇稿子的真实传播意图是什么。
- 它对我们有没有影响，影响哪类公关资产。
- 我们需要不应对、观察，还是进入判断中枢拍板。

不要把它做成「软文识别器」。软文只是其中一种可能，真正要判断的是：这篇稿是否改变了行业认知、竞品位置、媒体关系、话语体系或我们的解释成本。

## 先做事实核验

分析前先确认：

- 标题、作者、平台、发布时间、原始链接。
- 这是一篇原创、转载、通稿、访谈、评论、研究文章、政策通报，还是商业投放。
- 是否有同题材稿件同步出现。必要时搜索标题关键词、核心事件和关键句，判断它是不是矩阵传播的一部分。
- 关键事实是否来自单一信源。内部数据、细节规则、独家场景如果只服务某一方，通常不是偶然出现。

不能确认的内容要写清楚「未确认」，不要把猜测当结论。

## 判断顺序

### 1. 先给结论

第一段直接回答：

- 这篇稿大概率在替谁建立什么认知。
- 对我们是无关、观察，还是需要动作。
- 影响的是短期声量，还是长期信任、话语体系、媒体关系、业务解释成本。

### 2. 三个核心问题

**谁在推**

看谁被抬高、谁被压低、谁被省略。最高级的软文不一定点名夸自己，而是让读者自己得出「某一方更先进、更可靠、更有未来」的结论。

**为什么现在**

把发布时间放回现实节点：财报、政策、监管、竞品动作、行业事故、节假日、大促、融资、上市、组织变化。时机通常不是巧合。

**为什么这样说**

判断它为什么选择这个媒体、这个体裁、这个标题、这组数据和这套信源。体裁本身就是控制度信号：专访、独白、研究稿、评论、政策落地稿，对读者的影响路径不同。

### 3. 提炼核心断言

不要写摘要。要写它想让读者相信的 1-2 句话。

区分三层：

- 事实陈述：它明说了什么。
- 价值判断：它暗示谁更好、更先进、更负责任。
- 隐含预设：它默认什么标准是对的。

### 4. 看叙事动作

只写这篇稿实际使用的手法，不堆术语。

常见动作包括：

- 用词重新定义行业标准。
- 只放大有利数据，淡化不利条件。
- 用第三方、专家、研究员、用户故事降低防御。
- 用政策、技术、公益、AI、社会责任包装商业动作。
- 通过对比结构制造优劣感。
- 点名一方短板，同时回避另一方问题。
- 把企业动作嵌入更大的社会议题，借公共价值获得合法性。

### 5. 判断性质

给一个直接判断，不要绕：

- 中立报道：多方信源、事实增量明确、立场相对均衡。
- 有信息含量的软文：有真实信息，但叙事明显偏向某方。
- 纯软文：信息增量弱，主要服务单方形象。
- 批评性报道：主要揭问题，需判断是真调查、借题发挥，还是有推手。
- 政策协作类内容：企业数据被放进政策落地叙事里，公信力借用强，要看争议点是否被系统回避。

### 6. 判断对我们的影响

不要把所有竞对稿都写成威胁。只看它是否改变我们的工作条件。

重点判断：

- 是否直接点到我们。
- 是否创造了对我们不利的行业标准。
- 是否让竞对获得新叙事位置。
- 是否提高我们后续解释成本。
- 是否影响舆论资产、媒体关系资产、话语体系资产、内部判断资产。
- 是否和生意相关：用户心智、渠道承接、政策语境、资本市场、品类教育、风险治理。

结论分三档：

- **无需处理**：与我们关系弱，回应反而放大。
- **持续观察**：叙事有影响，但还没形成可行动威胁。
- **需要动作**：已影响资产或业务判断，需要回应、预案、发稿或判断中枢拍板。

## 输出结构

默认用这个结构：

```text
结论
一句话判断这篇稿在替谁建立什么认知，对我们是否需要动作。

一、这篇稿子在替谁建立什么认知
写传播方、目标受众、核心断言，不写摘要。

二、它为什么现在发
写时机逻辑，以及此刻发对传播方有什么好处。

三、它用了哪些叙事动作
只写识别到的关键手法，每条都要带原文依据或具体表现。

四、对我们的影响
判断影响等级、影响哪类公关资产、是否提高解释成本。

五、建议动作
不应对 / 观察 / 进入判断中枢。建议要具体，不撒网。
```

## 质量红线

- 不复述原文。
- 不把「可能」堆满全文。证据不足就写证据不足；能判断就给判断。
- 不用「框架设置」「议程设置」这类词当段落标题。
- 不把普通稿子抬成重大威胁。
- 不为了显得全面列无关部门。
- 不编造投放关系、媒体立场、作者动机。
- 不用「软文」两个字替代完整判断。

如果问题最后变成「我们要不要回应、反制、发稿、做预案、改业务动作」，转入 `judgment-hub` 收口。
